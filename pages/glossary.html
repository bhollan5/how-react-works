Glossary

This page contains a list of technical terms used on this website.

Terms are broken up into two sections, each organized alphabetically:

  • General webdev terms
  • React-specific terms









General webdev terms



Browser

A web browser is a standard software tool for rendering webpages from HTML, CSS and JavaScript.



CSS

CSS (Cascading Style Sheets) is a language for defining styles to apply to HTML elements.



DOM

The DOM, or Document Object Model, is a representation of a webpage's HTML code in Javascript objects.



Events

In programming, events typically called in response to user input, like keypresses or mouse clicks.



HTML

HyperText Markup Language, or HTML, is a markup language for describing content on a webpage.



Javascript framework

A JavaScript Framework is a set of JS tools that can be added to a website, to help make writing other scripts easier. ReactJS is a Javascript framework.



Javascript (web)

Javascript for the Web is an implementation of ECMAscript, with tools for interfacing with the Browser API.

This includes the document object, for accessing the DOM, and the window object.



Javascript (NodeJS)

NodeJS is an implementation of ECMAscript with tools for interfacing with an operating system.

This includes modules like fs for accessing the filesystem, and http, for running web servers.



Virtual DOM

A Virtual DOM is a Document Object Model that's managed by a JS framework, like React, instead of a web browser. In React, the VDOM is used to manage reactive state in composed components.









React-specific terms



Component

A React component is a set of HTML elements, with a name, state, and methods.



JSX

JSX is an extension of Javascript that allows programmers to define React components with simpler syntax.
